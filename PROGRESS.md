# 图片处理工具开发进展

## 文档日期
2026-04-26

## 当前定位

`Image Workbench` 是一个以 Windows 桌面工作流为主的本地图片后处理工具。
当前主入口是 `pywebview` 工作台，目标是把这些事情尽量放在同一个界面里完成：

- 单图处理
- 批量处理
- Pixiv 投稿辅助
- 日志观察与调试抓取

旧版 Tk GUI 仍然保留，但当前主要迭代都集中在 `pywebview` 版本。

## 当前项目结构

```text
image_processor/
```

- 输入目录 / 输出目录选择

- 浏览器自动投稿（Playwright）

- Pixiv 投稿预览

## 当前工作流

### 单图处理

1. 在首页加载一张图片
2. 在编辑页配置水印 / 打码 / 超分
3. 点击生成预览或导出结果
4. 如果开启 Pixiv，可直接从首页或发布页触发单图 Pixiv 流程

### 批量处理

1. 在发布页选择输入目录和输出目录
2. 配置处理顺序与 Pixiv 参数
3. 点击开始批量处理
4. 在日志区观察进度、错误与 Pixiv 上传信息

### Pixiv 调试

1. 先生成并保留当前图片的 Pixiv 草稿
2. 保持浏览器停留在 Pixiv 投稿页
3. 点击“抓取 Pixiv 调试快照”
4. 到 `tmp_pixiv_diag/` 查看 `.json`、`.html`、`.png`

## 当前已知限制

- Pixiv 浏览器投稿模式依赖 Pixiv 前端 DOM 结构，页面改版后仍可能需要重新适配。
- 批量模式下，如果选择浏览器手动确认，目前只支持单图；多图时应改用自动投稿。
- 自动化回归测试依赖运行环境中的实际依赖包；如果当前 Python 环境缺少 `Pillow` 等依赖，测试不会直接通过。
- `pywebview` 版本仍在持续迭代，旧版 Tk GUI 暂时只保留，不再作为主要开发目标。
- 全屏 / 窗口模式的工作台布局已做一轮适配，但后续仍需要继续用真实窗口截图校准细节。
